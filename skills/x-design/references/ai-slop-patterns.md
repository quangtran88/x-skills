# AI Generation Anti-Patterns

Common AI-generated UI clichés to avoid when implementing from a DESIGN.md. The site-specific Don'ts in section 7 always take precedence over this universal list.

Source: gstack design binary hardcoded rejection list + OpenAI "Designing Delightful Frontends" (Mar 2026).

## The List

1. **The 3-column feature grid** — icon-in-circle + bold title + description × 3
2. **Purple/violet gradient backgrounds** — the default "AI product" aesthetic
3. **Icons in colored circles** — generic placeholder for real visual design
4. **Centered-everything layouts** — no visual hierarchy, no tension
5. **Uniform bubbly border-radius** — same radius on every element regardless of function
6. **Decorative blobs, floating circles, wavy SVG dividers** — ornament without purpose
7. **Emoji as design elements** — substitutes for real iconography
8. **Colored left-border on cards** — the "sidebar accent" crutch
9. **Generic hero copy** — "Welcome to [X]", "The future of [Y]", "Supercharge your [Z]"
10. **Cookie-cutter section rhythm** — identical spacing/sizing for every section, no editorial pacing
