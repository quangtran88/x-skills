## How to use this catalog

1. User names a brand ("use Linear's style") → direct lookup by slug
2. User describes intent ("warm editorial") → match intent tags, propose 2–3 candidates
3. User asks "what styles are available?" → show a category they care about
4. Construct the raw URL by substituting `{commit}` and `{slug}` from the table below
5. `curl -fsSL <url> -o <target>/DESIGN.md` (target defaults to cwd)

Tags are derived from each site's one-liner — use them for fuzzy intent matching.

## Re-pinning to a newer commit

To update `pinned_commit` in `config.json`:

1. `curl -s https://api.github.com/repos/VoltAgent/awesome-design-md/commits/main | jq -r .sha` → new SHA
2. Check new site count: `curl -s https://api.github.com/repos/VoltAgent/awesome-design-md/contents/design-md | jq 'length'`
3. Update `config.json` → `pinned_commit`, `pinned_at`, `site_count`
4. If new slugs exist, add rows to the correct category in this catalog (or remove deprecated ones)
5. Re-verify a random raw URL to confirm the commit + path still resolves
