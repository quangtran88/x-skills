## Architecture at a Glance

```
User request
    │
    ├─── /x-do ──→ Mode A/B/C/D/E/F ──→ Dispatch to executor ──→ Verify ──→ Done
    ├─── /x-research ──→ Signal detection ──→ Tool/agent dispatch ──→ Synthesize ──→ Done
    ├─── /x-review ──→ Step 1-4 ──→ Verdict (approve/request changes)
    ├─── /x-bugfix ──→ Mode Q/A/B/C ──→ Investigate → Fix → Verify ──→ Done
    ├─── /x-design ──→ Slug resolution ──→ Fetch DESIGN.md ──→ Optional handoffs
    ├─── /x-api-pentest ──→ Step 1-6 ──→ Report (markdown + SARIF)
    ├─── /x-omo ──→ Agent/model dispatch ──→ Result
    ├─── /x-gemini ──→ Gemini CLI ──→ Result
    ├─── /x-qa ──→ Profile scan/run ──→ Per-test fanout ──→ Verdict envelope
    ├─── /x-team ──→ Decompose features ──→ Spawn workers ──→ x-qa gate ──→ Merge queue
    ├─── /x-worktree ──→ Branch + worktree ──→ Optional auto-isolation
    └─── /x-skill-improve ──→ Session analysis ──→ Alignment report ──→ Apply fixes
```

## Key Principles

1. **Router, not executor**: Skills classify and route. They do not execute complex work directly.
2. **Capability pinning**: Detect once at setup. Pin at bootstrap. Never re-check per dispatch.
3. **Slot resolution**: Pluggable dependencies resolved via 3-layer cascade (user → skill → default).
4. **Primitive discipline**: Every dispatch is `handoff` (sync) or `assign` (async fan-out).
5. **Cheapest-viable-first**: Free tools before token-billed before agent-billed.
6. **Verification-before-completion**: Long-running skills must run the completion cascade before claiming done.
