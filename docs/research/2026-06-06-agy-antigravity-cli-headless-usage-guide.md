# Part 1 â€” Headless Usage Guide (binary-verified)

## 1. Headless invocation (VERIFIED via `agy --help` + live runs)

```bash
agy -p "<prompt>"                         # --print: one-shot, prints response to stdout, then exits
agy --model "<name>" -p "<prompt>"        # per-call model selection (added in 1.0.5)
agy --add-dir <path> -p "<prompt>"        # mount a directory into the workspace (repeatable)
agy -c -p "<follow-up>"                   # --continue most recent conversation
agy --conversation <id> -p "<follow-up>"  # resume a specific conversation by id
agy --print-timeout <dur> -p "<prompt>"   # wait cap for print mode (default 5m0s, e.g. 120s)
agy --sandbox -p "<prompt>"               # terminal restrictions (now propagates in -p as of 1.0.6)
agy --dangerously-skip-permissions -p ... # auto-approve ALL tool/edit confirmations (YOLO)
agy --log-file <path> -p "<prompt>"       # override CLI log path
```

- **The prompt MUST be the value of `-p`/`--prompt`.** A bare positional prompt is silently ignored and agy emits a canned greeting (e.g. "I am currently using Gemini 3.1 Proâ€¦") â€” verified in Part 2. Always `agy -p "<prompt>"`, never `agy "<prompt>"`.
- **There is NO structured/JSON output.** Print mode emits **markdown/plain text only**. `--json` / `--output-format` do **not exist** (refuted: `flags provided but not defined: -json`). Any downstream parser must consume plain text.
- **Print mode buffers the whole response to the end** (no streaming). A hard `timeout` that fires mid-run loses *everything* (exit 124, 0 bytes). Budget `--print-timeout`/outer timeout generously (آ§12 saw this masquerade as a "hang").
- **Output chrome is prompt-controlled.** With a normal prompt agy may append a `### Work Summary` block; an explicit "Output ONLY the list, no preamble or summary" suppresses it (live-verified). A wrapper must either instruct-to-suppress or strip trailing chrome.
- **Permission hang is conditional, not universal.** Pure text/Q&A in `-p` returns fine **without** `--dangerously-skip-permissions` (live-verified). Read-only tools (Google Search, file reads via `--add-dir`) also auto-run. The hang risk is only when the agent wants **shell exec or file edits** in a non-TTY â†’ it blocks forever on a hidden Y/N. For a read-only advisor wrapper, do **not** pass `--dangerously-skip-permissions`; for autonomous build/test tasks, you must.

## 3. Google Search grounding + browser (VERIFIED live + binary protobuf)

- Grounding is **prompt/agent-driven, NOT a flag** (`--grounding` refuted). Including "Use Google Search" / "Use web search" in the prompt triggers it; the model also self-elects when the question needs fresh facts. Live test returned a correctly cited answer (Bun v1.3.14, source URLs).
- **Grounding is load-bearing for currency, not optional.** Part 2 (آ§14, finding 1) found that *without* an explicit web-search nudge, agy trusts the local repo over live docs and returned a **stale** identifier the repo still contained. For any "what's current / latest version / GA API" question, inject "Use Google Search to verify against current docs."
- The binary embeds a first-class `GoogleSearch` tool (`google_search` protobuf field) **and a full CDP/Chrome browser** (`cdp.Browser`, `BROWSER_OPEN`, `web_search_url`, `browser_page/text`) â€” so agy can *browse and scrape pages*, not just hit the Search grounding API. Richer than gemini CLI's search-only grounding.
- **Latency (reconciled with Part 2):** a *grounded web* flash-medium query took **105s** vs the gemini wrapper's **29s** (one sample) â€” the source of the "3â€“5أ—" figure. But a *scoped repo-grounding* run measured **79s vs 37s â‰ˆ 2أ—** (آ§13). So: ~2أ— for ordinary scoped repo research; 3â€“5أ— only for web-grounded/agentic-heavy runs (and far worse under the آ§14 pathologies). Budget `--print-timeout` accordingly.

## 9. Error handling in headless mode (VERIFIED behavior + prior-session finding)

- Success exits **0** with the response on stdout (verified). The hallucinated `429`/`130` exit codes are **not** real (`--bogus` control proves undefined-flag handling; no evidence of HTTP-style exit codes).
- **agy returns exit 0 even on failures** (prior session: exit 0 on "invalid project ID" / "not authenticated"; آ§14: exit 0 / exit 124 with empty stdout on traversal hangs). â†’ **Do not rely on exit codes for failure detection.** A robust wrapper must: (a) treat **empty/whitespace stdout** as failure, and (b) tail `--log-file` for failure markers â€” but **first strip the always-present auth noise** (آ§8) so it can't mislabel a generic empty failure as an auth error. Order the classification quota/planner â†’ narrow auth â†’ generic empty. This is the single biggest wrapper-hardening requirement.
- One transient empty-stream failure was observed on the *bare gemini* path (`-m gemini-2.5-flash`), not agy â€” but the empty-output failure mode exists on both; the wrapper's empty-stdout guard covers it.

## 10. Gotchas for headless / CI

1. **No JSON** â†’ parse plain text; strip ANSI (`colorScheme`) and any trailing `### Work Summary` chrome.
2. **Exit 0 lies** â†’ gate on empty-stdout + log-tail, never exit code (آ§9). Strip the آ§8 auth noise before classifying.
3. **Trust prompt** â†’ pre-seed `trustedWorkspaces` or the run hangs on first use of an untrusted dir.
4. **Permission hang** â†’ only when the agent wants shell/edit in non-TTY; pass `--dangerously-skip-permissions` for autonomous tasks, omit it for read-only advisory.
5. **Never `--add-dir` a large tree** â†’ آ§14: a repo-root `--add-dir` (65 GB monorepo w/ node_modules) hung agy 15â€“25 min, 0 bytes; a scoped 572 KB subtree returned in 79 s. Scope to the relevant subdirs.
6. **Serialize agy calls** â†’ آ§14: agy spawns a local gRPC language-server per invocation; concurrent calls contend and hang. Fan-out consumers (x-research Max Mode, x-qa) must run agy lanes sequentially.
7. **Latency** â†’ ~2أ— the gemini wrapper for a scoped grounded run (â‰ˆ79 s vs â‰ˆ37 s); 3â€“5أ— only for web-grounded/agentic-heavy runs (105â€“146s vs 29s). Budget `--print-timeout`; route bulk/no-tool work to a `(Low)` model.
8. **Prompt as flag value** â†’ `agy -p "<prompt>"`; a positional prompt is silently ignored (آ§1).
9. **Fast-moving target** â†’ 266 open issues, binary rebuilt the same day; pin behavior with live smoke tests, don't assume changelog == current.

## 14. Findings

**1. Only one lane got the web/docs anchor right â€” and it isn't a Gemini lane.** `explore` (gpt-5.4-mini) alone returned the GA event, by going to OpenAI docs / the upstream provider. Both Gemini lanes (agy + gemini) returned the **legacy** value that lives in the repo. agy even grepped, *confirmed* "it only matched the design doc," then reported it anyway. Lesson: for facts that may be stale in your own repo, the lane must be willing to leave the repo. agy's grounding is prompt-driven, so ANCHOR-1 is likely recoverable with an explicit "Use Google Search" nudge (آ§3) â€” worth a follow-up run.

**2. The repo anchor: everyone passed, via different paths.** agy and explore opened `coffer.ts` and quoted the literal definition; gemini gave the correct *call-shape* only because the replay prompt pre-loaded "writes go through dispatchCoffer (ACL)" â€” richer scaffolding than the original session that produced the `dispatchCoffer(driver,path,stream)` invention. gemini's reliability scales with how much ground truth you hand it; agy/explore derive it from the tool loop.

**3. explore surfaced facts the others did not.** Only explore reported (a) `dispatchCoffer` is **string-content only â€” it can't write raw `.webm` bytes** (a hard implementation blocker), and (b) an existing design memo that already rejects the two-file approach. agy critiqued CPU hazards and lifecycle well but missed the binary-write blocker; gemini (file-attach) couldn't reach either, structurally.

**4. agy DOES work â€” two wrong diagnoses, corrected by instrumentation.** agy returned 0 bytes across many runs. Diagnosis went (a) "timeout/non-viable" â†’ wrong; (b) "auth failure" (log screams `not logged into Antigravity`) â†’ wrong; (c) the truth: agy works â€” 79 s, exit 0, 9.5 KB, both anchors answered, three files opened â€” when run as **a single clean process with a scoped `--add-dir`**. Actual causes of the empties:
- **`--add-dir` on the 65 GB whole repo** blew up agentic traversal â†’ multi-minute hangs.
- **Concurrent agy processes** contend (per-call gRPC language-server). The "scoped also failed" run overlapped a 25-min whole-repo run; alone, the same scoped config returns in 79 s.
- The `not logged into Antigravity` lines appear verbatim on the *successful* run too â€” auxiliary caches, not serving (آ§8). The oauth token was valid throughout.

## 15. Recommendation

- **For grounded code/research today, prefer the agentic lanes (explore or agy) over gemini file-attach.** A smaller model that reads the files matched/beat the stronger model that can't. Keep gemini for *architecture critique* where you pre-load ground truth and treat every concrete identifier as unverified until grepped.
- **agy is a viable gemini-CLI replacement for repo-grounded research** â€” it grounds in the repo (gemini-agent can't) and is only ~2أ— slower (79 s vs 37 s), not the 3â€“5أ— feared. It ties gemini on the anchors and beats it on grounding capability.
- **agy's one gap is web/docs currency** (ANCHOR-1 legacy). For "what's the current API" questions, add "Use Google Search to verify against current docs" and re-run â€” likely closes the gap.
- **For the 2026-06-18 cutover, `bin/agy-agent` must:** scope `--add-dir`, serialize agy calls, and gate on empty-stdout + auth-noise-stripped log-tail (exit code lies). None are blockers â€” they're wrapper requirements, all folded into آ§11 and the task plan.
